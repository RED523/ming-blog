## Why

页脚已有「欢迎 👏🏻 你的访问」文案，但缺少可感知的站点热度反馈。在相同位置展示**全站累计访问次数**，能让读者感知博客被浏览的规模，并满足站长对简单流量展示的常见需求。

## What Changes

- 在页脚「欢迎 👏🏻 你的访问」**附近**展示**累计访问次数**（全站维度，非单页 PV）。
- 每次可被计为一次「访问」的触发规则、存储与防刷策略在 **design/spec** 中约定（例如按会话或按请求去重、部署环境约束）。
- 计数**递增**与**读取展示**需可靠；若依赖外部存储或托管能力，需在环境与密钥上可配置。
- 不改变页脚其余版权与导航结构；样式与现有 `Footer` 视觉层级协调。

## Capabilities

### New Capabilities

- `site-visit-counter`: 定义页脚累计访问次数的展示位置、计数语义、失败与降级表现，以及实现所需的可观测约束。

### Modified Capabilities

- （无）当前 `openspec/specs/` 下无既有能力规范需增量修改；仅新增 `site-visit-counter`。

## Impact

- **代码**：`src/components/Footer.tsx` 及可能的客户端封装、Route Handler 或 Server Action；若拆分组件则新增 `src/components/` 下小模块。
- **依赖与基础设施**：若采用 Redis/KV/托管计数 API，需新增相应 SDK 或 HTTP 调用及环境变量；纯自托管方案则影响部署文档。
- **隐私与性能**：计数请求应轻量，避免阻塞首屏；不向用户收集超出计数所需的个人信息。
